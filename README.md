# DevLab-One

DevLab-One is a startup software workspace designed for building products with human leadership and specialized AI agents.

## Workspace Structure
- `ai/`: company context, agent roles, prompts, standards, workflows, templates, knowledge, memory, and configuration.
- `apps/customer-portal/`: customer-facing web application.
- `apps/admin-portal/`: internal admin application.
- `apps/api/`: backend API services.
- `apps/mobile/`: mobile application.
- `apps/shared/`: shared code and reusable assets.
- `docs/`: architecture, API, deployment, and onboarding documentation.
- `scripts/`: automation and developer scripts.

## How to Use the AI Folder
1. Start with `ai/company/company.md` for mission and operating principles.
2. Use `ai/registry.md` to understand agent responsibilities.
3. Use `ai/prompts` for repeatable work instructions.
4. Use `ai/standards` before implementation and review.
5. Use `ai/workflows` to move work from idea to release.
6. Record decisions, lessons, sprint history, and known issues in `ai/memory`.

## Daily CEO Meeting
Use these files to run the daily company call:
- `ai/workflows/daily-ceo-meeting.md`
- `ai/templates/daily-ceo-meeting.md`
- `ai/prompts/daily-ceo-meeting.md`

The meeting should cover active project status, sprint health, blockers, new client or project ideas, tool needs, investment needs, company improvements, feedback, and AI-agent resource planning.

## Project Idea Discovery
Use these files when the Business Analyst team needs to generate new project ideas:
- `ai/workflows/first-project-idea-cycle.md`
- `ai/workflows/project-idea-discovery.md`
- `ai/templates/project-idea-brief.md`
- `ai/prompts/generate-project-ideas.md`
- `ai/memory/project-ideas.md`

## Startup Operating Rule
Move quickly in small, reviewable steps. Use AI agents to increase speed and quality, while keeping humans responsible for product direction, security, releases, and customer-impacting decisions.
