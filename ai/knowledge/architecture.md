# Architecture Knowledge

## System Overview
The workspace is organized as a multi-application software platform:

- `apps/customer-portal`: customer-facing web app.
- `apps/admin-portal`: internal admin web app.
- `apps/api`: backend API services.
- `apps/mobile`: mobile application.
- `apps/shared`: shared code, models, utilities, and design assets.

## Preferred Architecture
- Angular frontend applications.
- .NET or Node.js backend services.
- PostgreSQL database.
- REST APIs documented with OpenAPI.
- CI/CD deployments.
- Centralized logs and monitoring.

## Key Quality Attributes
- Security.
- Maintainability.
- Observability.
- Accessibility.
- Performance.
- Testability.
