# Naming Standard

## General
- Use names that reveal business meaning.
- Avoid unclear abbreviations.
- Keep naming consistent across UI, API, database, and docs.

## Files
- Use kebab-case for markdown and frontend files.
- Use project conventions for backend files.

## Code
- Variables and functions should describe intent.
- Boolean names should read as true or false, such as `isActive` or `hasAccess`.
- Avoid generic names like `data`, `item`, or `manager` unless the scope is obvious.

## Database
- Use singular or plural consistently across the project.
- Name foreign keys clearly, such as `customer_id`.
