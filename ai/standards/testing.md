# Testing Standard

## Test Types
- Unit tests for business rules and isolated logic.
- Integration tests for APIs, database behavior, and service boundaries.
- End-to-end tests for critical user journeys.
- Manual testing for exploratory and visual review.

## Expectations
- Tests must be repeatable.
- Test names should describe behavior.
- Cover success, failure, and edge cases.
- Add regression tests for bugs.

## Release Gate
- Critical tests must pass before release.
- Known test gaps must be documented.
