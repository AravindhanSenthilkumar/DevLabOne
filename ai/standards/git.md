# Git Standard

## Branches
- Use short, descriptive branch names.
- Recommended format: `type/short-description`.
- Examples: `feature/customer-login`, `fix/payment-timeout`.

## Commits
- Keep commits focused.
- Use clear messages that explain the change.
- Reference task or issue IDs when available.

## Pull Requests
- Include summary, test evidence, risks, and screenshots for UI changes.
- Request reviews from relevant owners.
- Keep pull requests small enough to review well.

## Merging
- Merge only after checks pass and review is complete.
