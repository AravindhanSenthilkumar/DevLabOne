# Angular Standard

## Structure
- Use standalone components where appropriate.
- Keep smart/container logic separate from reusable presentational components.
- Put API access in services.
- Keep feature code grouped by business capability.

## Components
- Use typed inputs and outputs.
- Handle loading, empty, error, and success states.
- Keep templates readable.
- Avoid large components that mix unrelated workflows.

## Forms
- Use reactive forms for complex forms.
- Validate both client-side and server-side.
- Show clear validation messages.

## Testing
- Test business behavior, not framework internals.
- Cover services, guards, validators, and important components.
