# Coding Standard

## General
- Use TypeScript or C# types deliberately.
- Prefer readable names over abbreviations.
- Keep functions focused on one reason to change.
- Validate external input.
- Keep errors actionable and safe.

## Reviews
- Every production change requires review.
- Tests should match the risk of the change.
- Refactors must preserve behavior unless explicitly scoped.

## Dependencies
- Add dependencies only when they reduce real complexity.
- Check license, maintenance, and security before adopting.
