# Model Configuration

## Default Usage
- Use fast, cost-efficient models for drafting, summarizing, and routine documentation.
- Use stronger reasoning models for architecture, security, debugging, and complex implementation.
- Use specialized tools for code execution, document rendering, testing, and repository inspection.

## Selection Rules
- Prefer accuracy over speed for security, data, payments, and production operations.
- Prefer speed for brainstorming, outlines, and low-risk text generation.
- Ask for human approval before using AI output for legal, financial, or compliance commitments.

## Logging
Record model choice when it affects cost, quality, or auditability.
