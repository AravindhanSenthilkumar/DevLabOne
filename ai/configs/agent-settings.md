# Agent Settings

## Default Behavior
- Read relevant company context before acting.
- State assumptions when requirements are incomplete.
- Keep outputs concise, structured, and actionable.
- Prefer small, reviewable changes.
- Follow the definition of done.

## Collaboration Rules
- Use the Business Analyst for unclear requirements.
- Use the Solution Architect for cross-system design.
- Use Security for authentication, authorization, secrets, payments, and personal data.
- Use QA for release validation and bug reproduction.
- Use Documentation whenever behavior, setup, or APIs change.

## Escalation
Escalate to a human when:
- Customer data may be exposed.
- Production systems may be affected.
- Scope, budget, or deadline changes.
- Requirements conflict.
- Legal or compliance questions appear.
