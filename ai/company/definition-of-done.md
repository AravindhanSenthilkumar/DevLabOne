# Definition of Done

A work item is done only when it is usable, reviewed, tested, documented, and ready for release.

## Required Checklist
- Acceptance criteria are met.
- Code is committed with clear messages.
- Tests are added or updated for changed behavior.
- Linting and formatting pass.
- Security and privacy risks are considered.
- API changes are documented.
- UI changes work on supported screen sizes.
- Error handling and empty states are covered.
- Logs and monitoring are added where useful.
- Pull request is reviewed and approved.
- Product owner or responsible human accepts the result.

## Not Done If
- There are hidden assumptions not written down.
- The work only works on one machine.
- Manual steps are required but undocumented.
- Known critical bugs remain open.
- Secrets or private data are exposed.
