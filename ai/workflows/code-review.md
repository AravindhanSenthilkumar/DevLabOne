# Workflow: Code Review

## Steps
1. Author opens pull request with summary, test evidence, and risks.
2. Code Reviewer checks correctness and maintainability.
3. Security reviews sensitive areas if needed.
4. QA verifies acceptance criteria for user-facing changes.
5. Author responds to required changes.
6. Reviewer approves when standards and definition of done are met.

## Review Focus
- Does it solve the right problem?
- Is it safe?
- Is it tested?
- Is it understandable?
- Can it be operated in production?
