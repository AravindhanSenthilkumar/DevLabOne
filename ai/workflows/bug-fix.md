# Workflow: Bug Fix

## Steps
1. QA or support records the bug with reproduction details.
2. Relevant engineering agent investigates root cause.
3. Business Analyst confirms expected behavior if unclear.
4. Developer makes the smallest safe fix.
5. QA verifies the fix and adds regression coverage.
6. Code Reviewer reviews the patch.
7. Documentation updates known issues or release notes if needed.

## Required Outputs
- Bug report.
- Root cause.
- Fix summary.
- Regression test.
