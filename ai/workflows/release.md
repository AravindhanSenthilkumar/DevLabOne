# Workflow: Release

## Steps
1. Project Manager confirms scope.
2. QA confirms test status.
3. Security confirms no blocking risks.
4. Documentation prepares release notes.
5. DevOps deploys to target environment.
6. Team runs smoke tests.
7. Monitoring is checked after deployment.
8. Known issues and follow-ups are recorded.

## Required Outputs
- Release checklist.
- Release notes.
- Deployment record.
- Post-release validation.
